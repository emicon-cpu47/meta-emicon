Name: deviceconfiguration\MapAlwaysIecAddress

Default Value: False

Recommended Value: False

XML Type: boolean

dotNet Type: System.Boolean

Example:

<ts:section name="deviceconfiguration">
      ...
    <ts:setting name="MapAlwaysIecAddress" type="boolean" access="visible">
        <ts:value>0</ts:value>
    </ts:setting>
      ...
</ts:section>

Documentation:

Type: Boolean, true or 1:
IoDrvReadInputs and IoDrvWriteOutputs are always copying to input %I or output %Q space. 
The mapping to the existing variables is done in functions afterreadinputs or beforewriteoutputs.
SysMem library must be included in the library manager because if a structured parameter is mapped SysMemCpy 
is used to transfer the data from % to the existing variable.
It the setting is enabled IO drives could optimize the coping of data in IoDrvReadInputs and IoDrvWriteOutputs 
by copying the complete data with one SysMemCpy because the IO-driver must not handle each mapping entry separately.
False or 0 (default):
Standard behavior as in old versions.
