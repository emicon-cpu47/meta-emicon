Name: visualization\defaultstyle

XML Type: string

dotNet Type: System.String

Example:

<ts:section name="visualization">
      ...
    <ts:setting name="defaultstyle" type="string" access="visible">
        <ts:value></ts:value>
    </ts:setting>
      ...
</ts:section>

Documentation:

Using this optional setting the default style can be set. This style will be used initially when a visualization manager is
inserted to applications below an according device. The value can either be 'none' to start without using a style, a full
name of a visualization style like 'styletitle, 1.2.3.4 (authoringcompany)' or a reference without an explicit version 
'styletitle,authoringcompany' to use the recent installed version of this style.
If this setting is not used, the style last chosen by the developer will be used in such a situation.

PlugIn:

VisualObject.plugin (3.5.11.0)
